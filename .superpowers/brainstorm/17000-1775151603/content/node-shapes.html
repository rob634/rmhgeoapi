Node Shape Language

Your YAML has 5 node types. Should each get a distinct shape, or keep it simple?

A

Distinct Shapes per Type

Each node type gets a unique shape — flowchart convention. Experienced users read the shape before the label.

Shape Legend

Pros

  • Type visible at a glance without reading labels
  • Follows flowchart conventions
  • Fan-out/fan-in visually indicate scatter/gather

Cons

  • 5 shapes to learn
  • Trapezoids harder to render + fit text into
  • More SVG complexity
B

Rectangles + Color + Icon

All nodes are rounded rectangles. Type communicated via fill color and a small icon/badge. Cleaner, easier to render, text always fits.

Color + Badge Legend

Pros

  • Uniform rectangles — text always fits
  • Simpler SVG rendering
  • Color carries the type, icon reinforces
  • Easier to add status overlays later

Cons

  • Less distinct at a quick glance than shapes
  • Color-blind users need the icon too (but icons help)

Recommendation: B (Rectangles + Color + Icon)

Uniform rectangles are much simpler to render and lay out with dagre. Text always fits (no squeezing labels into diamonds/trapezoids). The color + icon gives you type recognition without SVG path complexity. Status overlays (border color, background tint for completed/failed/running) layer cleanly on top of rectangles.
