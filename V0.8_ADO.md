## 3. PHASE 2: ADO STRUCTURE DESIGN

### 3.1 Feature Template (Manager-Friendly)

Each Feature (formerly Epic) in ADO:

```
Title: [Plain English name]
Description:
  WHAT: [1 sentence - what capability does this provide?]
  WHY: [1 sentence - what business problem does it solve?]
  STATUS: [Complete | In Progress | Planned]

Acceptance Criteria:
  - [Measurable outcome 1]
  - [Measurable outcome 2]

Tags: [geo-platform, v0.8, raster/vector/infra]
```

**Example**:
```
Title: Vector Data Pipeline
Description:
  WHAT: Transform uploaded shapefiles into queryable map layers
  WHY: Enables GIS analysts to publish data without ArcGIS Server
  STATUS: Complete

Acceptance Criteria:
  - Users can upload Shapefile, GeoJSON, GeoPackage
  - Data appears in OGC Features API within 5 minutes
  - Vector tiles render in web maps

Tags: geo-platform, v0.8, vector, complete
```

### 3.2 User Story Template

Each User Story (formerly Feature) in ADO:

```
Title: [Action-oriented name]
Description:
  AS A [role]
  I WANT [capability]
  SO THAT [benefit]

  IMPLEMENTATION NOTES:
  [Technical context for developers - optional for managers]

Acceptance Criteria:
  - [ ] [Testable criterion 1]
  - [ ] [Testable criterion 2]

Story Points: [1-13 Fibonacci]
```

### 3.3 Detail Level Strategy

| Status | Task Detail | Rationale |
|--------|-------------|-----------|
| **Complete** | Collapsed/none | Managers see "done", no clutter |
| **In Progress** | Full tasks visible | Shows activity, tracks progress |
| **Planned** | High-level tasks only | Enough for estimation, not over-specified |

**ADO Query for Managers**:
```
Work Item Type = 'Feature'
AND State IN ('Active', 'New')
ORDER BY Priority
```

This shows only active work with activity beneath it.
